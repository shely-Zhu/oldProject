wap组件、基本样式使用

基本结构及相关样式、js

1. mui卡片视图

结构：

<div class="mui-card marginTop centerWrap">
    ……
</div>

mui-card为卡片视图必需class，相关样式：白色背景，边框阴影
marginTop为上边距0.8rem，当该卡片需要上边距时，可以添加此class
centerWrap为没有左边距1.6rem时，即内容水平居中的时的class
相关样式在common/less/baseClass.less中

2. 表单

结构：

<div class="form-wrap">

    <div class="formList">
        ……
    </div>

    <div class="formList">
        ……
    </div>
</div>

form-wrap为表单区域，每一行表单元素为formList，根据左右是否有icon，是否有图文验证码、短信验证码等添加对应class，表单区域样式文件：common/less/mui/newFormMui.less，内有相关class说明。
每一表单元素公共结构放在common/views/form下，可直接使用。
js相关：
1） 校验文件：common/js/input.js，包括表单点击、校验、发送短信验证码、语音验证码、图文验证码等功能。
2） 每一表单元素所需校验的配置，在common/js/components/form/inputCheckSetting.js中，并被input.js引入；每种表单校验函数，在common/js/components/form/inputFunc.js中，并被inputCheckSetting.js引入，在inputCheckSetting.js中，同时将校验配置列表和校验函数exports出来
3）common/js/components/form/formAPI中的文件，为一些表单所需的数据，比如省市区联动的数据、实名认证银行列表数据的获取等等，并处理成mui-poppicker所需格式输出，以便表单点击进行选择时使用mui-poppicker实现模拟手机下方滚动弹出层选择的效果
4）短信验证码倒计时开始及结束的按钮状态变化，在common/js/components/form/timeCount.js中

3. 有> 或 向下箭头的可点击区域

结构：

<ul class="mui-table-view"> 
    <li class="mui-table-view-cell mui-collapse">
        <a class="mui-navigate-right" href="javascript:;">可点击可点击</a>
    </li>
</ul>

不加mui-collapse，箭头向右，加了mui-collapse，箭头向下。
a标签上有unable，表示不可点击。
a标签上有hasSelect，表示已选择。
样式文件：/common/less/mui/selectMui.less。

4. 列表展开收起

结构：

<div class="actionWrap">
    <div class="actionList">
        ……
    </div>
</div>
<div class="iconfont actionBtn beginAction">
</div>

actionWrap，标识当前是折叠区域
actionList，标识是折叠列表的每一条，可以是任意元素，元素内也可以是任意内容，目前wap中仅有银行卡列表可展开收起。
actionBtn，标识是展开收起按钮，beginAction用于js中使用，防止actionBtn与其他class冲突，但也可以不用

样式文件：common/less/components/openOrClose.less
js文件：common/js/components/openOrClose.js

5. 左右滑动区域

效果见公募产品列表页、私募产品列表页等。导航有不止一个标签，左右滑动可切换当前区域。
目前有两种方式实现：
1） 结构在页面中写死，见payThemeCash.html页面，js中监听$("#slider").on("slide",function(e){})方法，根据不同e.detail.slideNumber的值加载不同区域的数据
2）调用js插件common/js/components/tabScroll.js形成页面结构，见productList.html页面，js中初始化该插件并传参，设置滑动后的回调函数，加载不同区域的数据

6. 上拉加载

目前有两种方法实现：
1） 基于iscroll.js的上拉加载，调用插件common/js/components/slideFullPage.js，回到顶部按钮也根据iscroll.js的回到顶部实现，见我的资产页面
加载更多icon及文案：common/views/upLoad.html
2） 基于mui的上拉加载，见公募产品列表页，回到顶部按钮也基于mui实现
js初始化：

mui.init({
    pullRefresh: {
        container: '', //上拉加载区域
        up: {
            contentrefresh: '拼命加载中',
            contentnomore:'没有更多了',//可选，请求完毕若没有更多数据时显示的提醒内容；
            callback : function(){
                //执行ajax请求及对数据的处理               
            }
        }
    }
});
//触发上拉加载的callback函数
mui('.contentWrapper').pullRefresh().pullupLoading();

回到顶部按钮的js文件：common/js/components/goTopMui.js

9. 常见黄色大按钮

结构：

 1）当按钮上下左右有外边距时，如表单下方的提交按钮，使用此结构：
 <div class="btnButton">
      <button type="button" class="mui-btn"></button>
 </div>
 btnButton提供外边距margin

 2）当按钮不需要外边距时，如固定在页面底部的按钮，使用此结构：
 <button type="button" class="mui-btn"></button>
 class说明：
 若按钮固定在底部，在 mui-btn 上添加 bottom 
 若按钮置灰，在 mui-btn 上添加 disable
 若按钮上有icon，在对应页面自己的less文件中，重置该按钮 mui-btn 的 before 样式的content: ''

样式文件：common/less/baseClass
